## What is RustHound TUI?

A **terminal-native** BloodHound CE companion, written in Rust. Point it at a SharpHound ZIP — or a loose folder of JSON — and it instantly:

> Parses and summarizes every collection type: users, groups, computers, domains, GPOs, OUs, containers, **and full ADCS objects** (cert templates, CAs, NTAuthStores, issuance policies)
>
> Resolves bare usernames — `--from JOHN` just works, no `@DOMAIN` required, even across multiple domains
>
> Lets you walk the graph interactively through `tree`, `paths`, or a complete **ratatui** terminal UI
>
> Finds attack paths to Tier Zero without spinning up Neo4j or opening a browser
>
> Runs comfortably on a 4GB laptop where BloodHound CE OOMs on `default password change`

## Quick start

```bash
# Clone and build
git clone https://github.com/yourname/rusthound-tui
cd rusthound-tui
cargo build --release

# Parse a SharpHound ZIP and print a summary
./target/release/rusthound-tui ingest collection.zip

# Tree from a user — bare name, no @domain needed
./target/release/rusthound-tui tree collection.zip --from JOHN --depth 4

# Attack paths to Tier Zero
./target/release/rusthound-tui paths collection.zip --from ALFRED

# Full interactive terminal UI
./target/release/rusthound-tui tui collection.zip

# Deep analysis report
./target/release/rusthound-tui analyze collection.zip
```

### `tree`

Print an interactive attack-path tree rooted at a node. Edges are color-coded by kind (red for `GenericAll`, yellow for `AdminTo`, blue for `MemberOf`, and so on). Supports bare-name resolution — no `@domain` required.

```text
rusthound-tui tree <PATH> [OPTIONS]
```

| Option | Description | Default |
|---|---|---|
| `-f`, `--from` | Start node — name or object ID | First real user |
| `-d`, `--depth` | Tree depth | `3` |
| `-a`, `--attack` | Show attack edges only | off |
| `--max-children` | Max children per node before `…` | `25` |
| `--tier-zero` | List all Tier Zero objects instead | off |

```text
# Explore from a specific user
$ rusthound-tui tree collection.zip --from JOHN

ADMINISTRATOR@CERTIFICATE.HTB  [User] [admincount]
├── Owns ──► SIGNEDUSER@CERTIFICATE.HTB [CRT ADCS]
├── WriteDacl ──► SIGNEDUSER@CERTIFICATE.HTB [CRT ADCS]
├── GenericWrite ──► SIGNEDUSER@CERTIFICATE.HTB [CRT ADCS]
├── MemberOf ──► DOMAIN ADMINS@CERTIFICATE.HTB [Group] [admincount] *
├── MemberOf ──► ENTERPRISE ADMINS@CERTIFICATE.HTB [Group] [admincount] *
└── MemberOf ──► ADMINISTRATORS@CERTIFICATE.HTB [Group] *

Graph: 332 nodes  1152 edges — use --depth N for deeper traversal
```

```text
# Learn what connects to a node
$ rusthound-tui tree collection.zip --from "SIGNEDUSER@CERTIFICATE.HTB"

SIGNEDUSER@CERTIFICATE.HTB [CRT ADCS]
├── Owns ◄── ADMINISTRATOR@CERTIFICATE.HTB [User] [admincount]
├── WriteDacl ◄── ADMINISTRATOR@CERTIFICATE.HTB [User] [admincount]
└── GenericWrite ◄── ADMINISTRATOR@CERTIFICATE.HTB [User] [admincount]
```

```text
# All Tier Zero / high-value nodes
```

### `paths`

Find BFS shortest attack paths from a node to every reachable high-value target. Stops at the first hit per target and shows edge kinds between each hop.

```text
rusthound-tui paths <PATH> -f <NAME> [--max N]
```

```text
$ rusthound-tui paths collection.zip --from JOHN

Attack paths from: JOHN@TOMBWATCHER.HTB

Found 1 attack path(s):

Path 1:

  │
  CanPSRemote
  │
  DC01.TOMBWATCHER.HTB [Computer]
  │

  ENTERPRISE DOMAIN CONTROLLERS@TOMBWATCHER.HTB [Group]
  │
  GetChangesInFilteredSet
  │
  TOMBWATCHER.HTB [Domain] * HIGH VALUE
```

## Features

| Feature | Details |
|---|---|
| **Bare name resolution** | `--from JOHN` works. RustHound auto-resolves `JOHN` to `JOHN@DOMAIN` by splitting on `@`. If two `JOHN`s exist in different domains, it shows both and lets you pick. |
| **ADCS objects** | Certificate templates, root CAs, enterprise CAs, NTAuthStores, AIA, issuance policies — all parsed and inserted into the graph with their ACEs. ADCS nodes render as `[CRT ADCS]` in the TUI. |
| **ZIP + loose JSON** | Drop a SharpHound ZIP file, a directory of JSON files, or a single JSON file. Auto-detected, no flags needed. |
| **Interactive TUI** | ratatui-powered three-panel browser. Search by name or kind. Navigate outgoing/incoming edges. Drill into any node. |
| **Attack paths** | BFS from any node to all reachable high-value targets, with edge kinds shown per hop. |
| **Offline analysis** | Kerberoastable, AS-REP roastable, and unconstrained delegation detection, plus a full ACE breakdown and JSON export. |
| **Tree explorer** | Color-coded adjacency tree with an attack-edge filter, depth control, and max-children limits. |
| **Low memory** | Runs on 4GB machines where BloodHound CE itself OOMs. No Neo4j. No Electron. |

## Architecture

```
                    ┌─────────────┐
                    │ SharpHound  │
                    │   ZIP/JSON  │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   ingest    │
                    │  (zip +     │
                    │  serde)     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ ParsedDataset│
                    │  users,     │
                    │  groups,    │
                    │  computers, │
                    │  adcs, …    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────────┐
                    │ graph_builder   │
                    │  (petgraph BFS) │
                    │  NodeKind,      │
                    │  ACE edges,     │
                    │  name index     │
                    └──────┬──────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼─────┐ ┌───▼───┐ ┌─────▼─────┐
       │   tree     │ │ paths │ │   tui     │
       │ (color-cld)│ │ (BFS) │ │ (ratatui) │
       └────────────┘ └───────┘ └───────────┘
```

Five modules, one binary. The heavy lifting is:

- **`ingest`** — SharpHound JSON deserialization with serde (7 core + 6 ADCS collection types)
- **`ad`** — Typed AD object models with property accessors
- **`graph_builder`** — Converts parsed data into a digraph with named ACE edges, node kinds, and a case-insensitive name index
- **`tree_view`** — Recursive tree printer plus a BFS attack-path finder
- **`tui`** — ratatui three-panel interactive browser

## Why not BloodHound CE?

| | BloodHound CE | RustHound TUI |
|---|---|---|
| **RAM at idle** | ~1.5 GB (Neo4j) | ~10 MB |
| **RAM at peak** | OOM on 4GB | ~200 MB |
| **Setup** | Docker / Neo4j + config | Single binary |
| **UI** | Electron web app | Terminal (ratatui) |
| **Offline** | No | Yes — parse + analyze without any server |
| **Bare names** | Always needs `@domain` | Auto-resolves |
| **ADCS objects** | Full ESC edges | Raw objects + ACEs (ESC logic not yet added) |

## Philosophy

RustHound TUI is built for the laptop you actually bring to an engagement — the one with 4GB of RAM, a throbbing fan, and no Docker. It skips everything that needs a server (Neo4j, gRPC, GraphQL) and keeps only what fits in a single `cargo build` deploy: ingest, graph, explore.

**Still on the roadmap:**

- Synthetic ESC composite edges (template + CA + enrollment-rights analysis) — for answering "can I ESC1 to DA"
- A `--to` flag on `paths` for a direct target without the high-value filter
- ADCS raw-data export — the schema is already parsed, the properties bag is already there, it just needs a `--dump` flag
